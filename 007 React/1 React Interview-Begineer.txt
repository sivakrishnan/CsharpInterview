2Ô∏è1)What is Re-rendering in React?

React calling your component function again to calculate what the UI should look like.

It means:

	Component function runs again
	New Virtual DOM is created
	React compares old vs new (using Reconciliation Algorithm)
	Only changed parts update in real DOM
	
üî• What Actually Happens During Re-render?

Let‚Äôs say you have:

function Counter() {
  const [count, setCount] = useState(0);

  console.log("Component Rendered");

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </div>
  );
}

When you click button:

setCount() updates state
React calls Counter() again
New Virtual DOM created
React compares with previous version (using Reconciliation Algorithm)
Only <p> text updates in real DOM

üéØ When Does Re-render Happen?

‚úÖ 1Ô∏è) State changes
‚úÖ 2Ô∏è) Props change
‚úÖ 3Ô∏è) Context value changes
‚úÖ 4Ô∏è) Parent re-renders


üß™ Important: Function Runs Again

On every re-render:

	All variables re-created
	All calculations re-executed
	Functions recreated

That‚Äôs why performance optimization matters.

üß† Real Interview-Level Explanation

Re-render has 2 phases:

1Ô∏è) Render Phase

	Component function runs
	Virtual DOM created
	useMemo runs here

2Ô∏è) Commit Phase

	React updates actual DOM
	useEffect runs here
