1Ô∏è3) What is useRef?

useRef is a React Hook that Stores a mutable value that persists across renders without causing re-render when updated.

‚úÖ Syntax

		const myRef = useRef(initialValue);

It returns an object like:

		{
		  current: initialValue
		}

You access value using:

		myRef.current

üî• Two Main Uses of useRef

1Ô∏è) Accessing DOM Elements (Most Common)
2Ô∏è) Storing Values Without Re-rendering

1Ô∏è) Accessing DOM Elements (Most Common) Example: 

import { useRef } from "react";

function Example() {
  const inputRef = useRef(null);

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <>
      <input ref={inputRef} />
      <button onClick={focusInput}>Focus</button>
    </>
  );
}

2Ô∏è) Storing Values Without Re-rendering

function Example() {
  const renderCount = useRef(0);

  renderCount.current++;

  console.log("Rendered:", renderCount.current);

  return <div>Hello</div>;
}

Even though renderCount.current changes, it does NOT cause re-render.

1Ô∏è7) What is useCallback?

Memoizes a function to prevent unnecessary re-renders.

useCallback is a React Hook that memoizes a function and returns the same function reference unless its dependencies change, 
helping prevent unnecessary re-renders in child components.

With useCallback (Correct Way)

Modify Parent:

import React, { useState, useCallback } from "react";
import Child from "./Child";

function Parent() {
  const [count, setCount] = useState(0);

  const handleClick = useCallback(() => {
    console.log("Button clicked");
  }, []);

  console.log("Parent rendered");

  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count + 1)}>
        Increase Count
      </button>

      <Child onClick={handleClick} />
    </div>
  );
}

export default Parent;


Child Component: 

import React from "react";

function Child({ onClick }) {
  console.log("Child rendered");

  return (
    <button onClick={onClick}>
      Child Button
    </button>
  );

2Ô∏è1)What is Re-rendering in React?

React calling your component function again to calculate what the UI should look like.

It means:

	Component function runs again
	New Virtual DOM is created
	React compares old vs new (using Reconciliation Algorithm)
	Only changed parts update in real DOM
	
üî• What Actually Happens During Re-render?

Let‚Äôs say you have:

function Counter() {
  const [count, setCount] = useState(0);

  console.log("Component Rendered");

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </div>
  );
}

When you click button:

setCount() updates state
React calls Counter() again
New Virtual DOM created
React compares with previous version (using Reconciliation Algorithm)
Only <p> text updates in real DOM

üéØ When Does Re-render Happen?

‚úÖ 1Ô∏è) State changes
‚úÖ 2Ô∏è) Props change
‚úÖ 3Ô∏è) Context value changes
‚úÖ 4Ô∏è) Parent re-renders


üß™ Important: Function Runs Again

On every re-render:

	All variables re-created
	All calculations re-executed
	Functions recreated

That‚Äôs why performance optimization matters.

üß† Real Interview-Level Explanation

Re-render has 2 phases:

1Ô∏è) Render Phase

	Component function runs
	Virtual DOM created
	useMemo runs here

2Ô∏è) Commit Phase

	React updates actual DOM
	useEffect runs here
